# /commit - 会話コンテキストを考慮したコミット作成

以下の手順でコミットを作成してください。

## 1. ブランチ確認

現在のブランチを確認してください（`git branch --show-current`）。

main/masterブランチにいる場合:
- 変更内容から適切なブランチ名を提案してください（例: `feat/add-fish-alias`, `fix/nvim-config`）
- ユーザーに確認してから `git checkout -b <branch>` で新しいブランチを作成してください
- ユーザーがmainで続行したい場合はそのまま進めてください

## 2. 変更内容の確認

- `git status` で変更ファイルを確認
- `git diff` と `git diff --staged` で差分を確認
- ステージングされていない変更がある場合、ユーザーにどのファイルを含めるか確認

## 3. コミットメッセージの作成

**今回の会話の内容を考慮して**、Conventional Commits形式でメッセージを作成してください:
- `feat:` 新機能
- `fix:` バグ修正
- `docs:` ドキュメント変更
- `chore:` ビルド・ツール変更
- `refactor:` リファクタリング

メッセージは日本語でも英語でも、変更内容に適した方を選んでください。

## 4. コミット実行

- コミットメッセージの案をユーザーに提示
- 承認を得てからコミットを実行
- `git log --oneline -1` で結果を確認
